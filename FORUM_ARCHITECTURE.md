# Forum Architecture

## Цель

Публичный форум доступен без авторизации и показывает карточки изделий из основного каталога:

- табаки;
- кальяны;
- чашки;
- калауды;
- уголь;
- электрочаши.

Карточка изделия строится из справочников проекта, а пользовательский контент хранится отдельно в `forum`-схеме PostgreSQL.

## Модель данных

### Источник каталога

Базовые сущности форума не дублируют справочники:

- название;
- бренд;
- модель;
- параметры;
- наличие;
- системные теги.

Эти данные продолжают жить в `catalog` и `equipment`, а форум накладывает поверх них контентный слой.

### Контентный слой

В `forum` используются таблицы:

- `forum.item_topics` — карточка обсуждения для конкретного элемента каталога;
- `forum.comments` — короткие сообщения в обсуждении;
- `forum.reviews` — подробные отзывы с оценкой по 5-балльной шкале;
- `forum.comment_assets` — изображения комментариев;
- `forum.review_assets` — изображения отзывов.

`forum.item_topics` связывает форум с изделием через пару:

- `section_key`
- `reference_item_id`

Такой подход позволяет не копировать предметные сущности и не плодить отдельные справочники только ради форума.

## Media

Изображения продолжают жить в `media.assets`, а форум использует их через связи:

- `cover_asset_id` в `forum.item_topics`;
- таблицы `forum.comment_assets`;
- таблицы `forum.review_assets`.

Это оставляет единый media-конвейер:

1. загрузка файла через presigned URL;
2. регистрация `media.assets`;
3. привязка asset к карточке, комментарию или отзыву.

## API

Публичный API форума читает:

- справочники из `PlatformDataService`;
- форумный контент из PostgreSQL через `ForumService`.

Если `DATABASE_URL` отсутствует, используется fallback-режим с вычисляемым demo-контентом. При наличии PostgreSQL форум работает на реальных таблицах.
